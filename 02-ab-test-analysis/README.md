# 🧪 Проект 2: A/B Тестирование (Conversion Rate)

## Задача
Проверить, даёт ли новая версия страницы (Variant B) статистически значимый прирост конверсии по сравнению с текущей (Variant A).

## Данные
- 10,000 пользователей (5,000 на вариант)
- 2 недели теста
- Метрики: конверсия, выручка, ARPU

## Статистический метод
**Z-тест для пропорций** — стандарт для A/B тестов бинарных исходов (конверсия: да/нет).

Формула:
```
z = (pB - pA) / SE
SE = sqrt(p_pooled * (1 - p_pooled) * (1/nA + 1/nB))
p_pooled = (convA + convB) / (nA + nB)
```

## Что классический аналитик делает 3 часа
1. Выгружает данные из BI/базы
2. Считает конверсию в Excel
3. Ищет онлайн-калькулятор z-теста или пишет формулу вручную
4. Проверяет MDE, power, CI
5. Пишет выводы

## Что делает Augmented Analyst за 20 минут
1. **Агент генерации**: синтетические данные с известным ground truth (B лучше A на ~25%)
2. **Агент статистики**: автоматический z-test, CI, MDE, power
3. **Агент интерпретации**: выводы на русском + рекомендации для бизнеса
4. **Евгений**: проверяет корректность метода, формулирует бизнес-рекомендацию

## Ключевые результаты (примерные)
| Метрика | Variant A | Variant B | Разница |
|---------|-----------|-----------|---------|
| Конверсия | 12.3% | 15.3% | **+24.6%** |
| 95% CI | [11.4%, 13.2%] | [14.3%, 16.3%] | — |
| P-value | — | — | **0.000012** |
| ARPU | $10.40 | $13.31 | **+28.0%** |

## SQL-шаблоны
```sql
-- Базовые метрики по вариантам
SELECT
    variant,
    COUNT(*) as users,
    SUM(converted) as converters,
    ROUND(100.0 * SUM(converted) / COUNT(*), 2) as conversion_rate,
    SUM(revenue) as total_revenue,
    ROUND(1.0 * SUM(revenue) / COUNT(*), 2) as arpu
FROM ab_test_results
GROUP BY variant;
```

## Валидация агента
Мы **знаем ground truth**: при генерации B был настроен на 14.5% конверсии против 12% у A. Агент корректно выявил разницу и подтвердил статистическую значимость. Это демонстрирует, что методика работает на контролируемых данных.

## Инсайты
- Разница **высоко значима** (p < 0.01) — можно раскатывать на 100% трафика
- Прирост ARPU 28% превышает прирост конверсии → B не только конвертирует больше, но и привлекает более платёжеспособных пользователей
- MDE = 1.29% — тест был достаточно мощным для детектирования даже меньших различий
