## 2. Почему это быстрее классического подхода

### Сравнение: задача «Проанализировать воронку продаж»

| Этап | Классический аналитик | Augmented Analyst (мы) |
|------|------------------------|------------------------|
| **Понять задачу** | 15 мин разговор с PM | 5 мин, Евгений формулирует |
| **Выгрузить данные** | 30–60 мин SQL + ожидание | **0 мин** — агент генерирует или пишет оптимальный запрос |
| **Очистка/препроцессинг** | 1–2 часа вручную | **5 мин** — агент автоматически |
| **Анализ** | 1–2 часа группировки, метрики | **10 мин** — параллельные агенты |
| **Визуализация** | 30–45 мин в BI/Excel | **5 мин** — Plotly/HTML автоматически |
| **Выводы** | 30–45 мин формулировка | **5 мин** — агент генерирует драфт, Евгений редактирует |
| **Документация** | 30 мин | **0 мин** — автосохранение в Vault |
| **Итого** | **4–6 часов** | **30–40 минут** |

**Ускорение: 6–10x на типовых ad-hoc задачах.**

## 3. Почему это надёжнее классического подхода

### Проблема одиночного аналитика
- Усталость → ошибки в SQL (лишний JOIN, пропущенный фильтр)
- Когнитивные искажения → видит только то, что ищет
- Забывает проверить edge cases
- Не документирует промежуточные шаги

### Преимущества Human+AI
1. **Параллельная валидация**: один агент считает метрики, второй проверяет их другим методом, третий ищет аномалии
2. **Нет усталости**: агент просчитает 10 вариантов группировки так же быстро, как 1
3. **Автодокументация**: каждый шаг сохраняется в Vault — можно воспроизвести через месяц
4. **Проверка на контролируемых данных**: мы тестируем методику на синтетике с известным ground truth, прежде чем применять к реальным данным

## 4. Ключевые принципы работы

### Принцип 1: Human-in-the-Loop
Агент **не принимает решений**. Он готовит варианты, считает цифры, генерирует код. Евгений выбирает, валидирует, интерпретирует для бизнеса.

> «ИИ — это калькулятор нового поколения. Он считает быстро, но вы решаете, какая задача worth solving.»

### Принцип 2: Subagent-ферма
Сложная задача декомпозируется на параллельные подзадачи:
- Агент-SQL: оптимизирует запросы
- Агент-Python: строит модели и графики
- Агент-статистик: проверяет значимость
- Агент-писатель: формулирует выводы

### Принцип 3: Second Brain = Persistent Memory
Всё сохраняется в **Obsidian Vault**:
- SQL-шаблоны для повторного использования
- Python-скрипты с параметризацией
- Результаты анализов с датами
- Инсайты и гипотезы

Это значит: задача, решённая сегодня, решается **в 10 раз быстрее** через месяц — достаточно скопировать шаблон и поменять дату.

### Принцип 4: Автономное обучение
Агент **учится на задачах Евгения**:
- Запоминает предпочтительные визуализации
- Адаптирует SQL под конкретную схему БД компании
- Знает, какие метрики Евгений считает первыми
- Накапливает бизнес-контекст (что такое "хороший LTV" для этой компании)

## 6. Как это звучит для работодателя

> *«Я работаю как оператор агентской ИИ-системы, где мой AI-ассистент (Hermes Agent) параллельно генерирует SQL-запросы, строит статистические модели, визуализирует данные и документирует выводы. Это позволяет мне фокусироваться на стратегии и коммуникации с бизнесом, а рутинную аналитику выполнять в 6–10 раз быстрее. При этом я сохраняю полный контроль: каждый результат валидирую сам, а все методологии и шаблоны аккумулируются в живой базе знаний (Obsidian Vault), которая обучается вместе со мной.»*

## 7. Чем это отличается от «просто использования ChatGPT»

| | ChatGPT / Claude | Human + AI (Hermes Agent) |
|---|---|---|
| **Контекст** | Забывает разговор через 10 минут | Помнит Vault, проекты, предпочтения месяцами |
| **Инструменты** | Только текст | Может запускать Python, SQL, cron, парсить API, писать файлы |
| **Автономность** | Ждёт промпта | Может работать по расписанию (cron), реагировать на webhook |
| **Параллелизм** | Один диалог | Множественные subagents для разных задач |
| **Валидация** | Человек копипастит | Агенты перекрёстно проверяют друг друга |
| **Документация** | В истории чата | Структурированный Vault с тегами и связями |

|-----------|------|

| **Telegram** | Интерфейс взаимодействия (where applicable) |

## 9. Следующие шаги (роадмап)

1. **Подключение к реальным БД** — PostgreSQL/ClickHouse для live-аналитики
2. **Интерактивные дашборды** — Plotly Dash / Streamlit вместо статических отчётов
3. **Продвинутый ML** — sklearn/XGBoost для прогнозирования, не только EDA
4. **NLP-аналитика** — агент читает отзывы пользователей, выделяет темы и сентимент
5. **Автоматические alert'ы** — «метрика X упала на 15% → агент исследует причину → готовит драфт ответа PM'у»
